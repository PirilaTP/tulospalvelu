# 10.4 Karsintakilpailuun perustuva loppukilpailu

### 10.4 Karsintakilpailuun perustuva loppukilpailu

Loppukilpailun muodostaminen vastaa monessa suhteessa
menettelyä, jota on kuvattu edellä hiihdon sprintin menettelynä. Päällekkäisten
numeroiden välttäminen on kuitenkin varmistettava siirtämällä loppukilpailuvaiheen
numerot numerointikaavakkeella sopivalle alueelle, ellei sarjoja
ole jo määritelty niin, että päällekkäisyyttä ei voi syntyä.

Itse loppukilpailu muodostetaan sitten kaavakkeella *Loppukilpailun muodostaminen*. Kaavake on valmisteltu
ajatellen, että vaiheita on kaksi, ensin karsinta ja toisena loppukilpailu.
Kaavaketta käytetään karsintakilpailun tulospalvelutilassa.

Loppukilpailujen muodostamisen avuksi voidaan laatia
skriptitiedosto (oletusnimi `finsaannot.txt`),
joka
koostuu seuraavan kaltaisista riveistä:

```
A-FINAALI:; sarja;lkm/kars.sarja;kaikkiloput;annalähtöajat;ens.lähtö;lähtöväli;aalto;numeroalku;rajaus;takaa-ajo
karsintasarja
LOPPU
```

Tai esimerkkinä

```
A-FINAALI:; H21A; 10; 0; 1; 15:00:00; 60; 1; 1; 1; 0
B-FINAALI:; H21B; 10; 0; 1; ; ; 1; 201; 1; 0
C-FINAALI:; H21C; 0; 1; 1; 13:20:00; 60; 4; 0; 3; 0
H21K1
H21K2
H21K3
H21K4
LOPPU
A-FINAALI:; D21A; 10
B-FINAALI:; D21B; 0; 1; 1; 13:00:00; 60; 4; 0; 3; 0
D21K1
D21K2
LOPPU
```

Tiedoston rakenne muistuttaa läheisesti tietojen sijoittumista itse kaavakkeelle.
Kaksoispisteet sanan FINAALI jälkeen ovat pakollisia. Kenttien väleissä
on oltava erotinmerkki, joka voi olla joko puolipiste tai tabulaattori,
välilyönnit ovat sallittuja, mutta eivät toimi erotinmerkkeinä.

Rivi, joka alkaa edellä
`A-FINAALI:` sisältää samat kentät samassa
järjestyksessä kuin kaavake. Ruksattavat tiedot ilmaistaan numeroarvoilla 0 tai
1, muut tiedot samalla tavalla kuin kaavakkeella. Alussa voi kirjaimen
`A` paikalla
olla  `B, C, D` tai `E`.

Finaalisarjoista on annettava minimissään sarjan nimi ja
siihen poimittavien lukumäärä tai seuraavaan sarakkeeseen merkki, että mukaan
otetaan kaikki loput. Loput kentät voi jättää pois, jolloin ohjelma käyttää joko
sarjatiedoista saatuja tietoja (lähtöaika, lähtöväli ja numeroinnin alku) tai
oletusarvoja (annetaan lähtöajat, lähtöaallon koko on 1, mukaan otetaan vain
hyväksytyt, takaa-ajo ei ole käytössä).

Kaksi erotinmerkkiä, joiden välissä on korkeintaan
välilyöntejä tarkoittaa, että kyseiselle kentälle käytetään edellä kuvattua
oletusarvoa. Täten voidaan esimerkiksi käyttää sarjatietojen mukaista lähtöaikaa
ja lähtöväliä, mutta ilmoittaa numeroinnin alku, kuten yllä B-finaalin H21B
kohdalla on tehty. Naisten A-finaalin tiedot edustavat vähintä, mikä on
määriteltävä.

Merkki `takaa-ajo` voi
olla 0 tai 1 (1 tarkoittaa takaa-ajoa) ja `rajaus` 1, 2 tai 3, missä 1 tarkoittaa vain
hyväksyttyjä, 2 hyväksyttyjä, hylättyjä ja keskeyttäneitä ja 3 kaikkia
ilmoittautuneita ml. ei-lähteneet.

Finaalisarjojen jälkeen tulevat karsintasarjojen nimet
kukin omalla rivillään ilman muita tietoja.

Kunkin muodostusvaiheen lopuksi tulee rivi `LOPPU`, jonka jälkeen voi alkaa seuraavaa vaihetta
kuvaavien rivien ryhmä. Ohjelma lukee aina yhden vaiheen kerrallaan, minkä
jälkeen vaiheen tietoja voidaan tarvittaessa muokata ja vaihe sitten toteuttaa
ennen seuraavan vaiheen lukemista. Kun kaikki vaiheet on käyty läpi tai kaavake
suljettu ja avattu uudelleen, voidaan valita sama tai toinen skriptitiedosto
käsiteltäväksi. Vaihe voidaan ohittaa jättämällä toteutus väliin ja lukemalla
seuraava vaihe.

Skripti voidaan luoda ohjelmalla täyttämällä kutakin
vaihetta koskeva
kaavake ja kirjoittamalla sen sisältö skriptiin valikkovalinnan kautta. Kirjoittaminen tapahtuu
valitun tiedoston loppuun, joten se voidaan tehdä useassa vaiheessa. Toisaalta
on vanha skripti poistettava tai uusi laadittava eri nimellä, jos halutaan jättää
aiempi sisältö pois. Jos ennen skriptiin kirjoittamista tyhjennetään lähtöaika, lähtöväli,
aallon koko ja/tai numeroinnin alkuarvo, määräytyvät poistetut tiedot sarjatiedoista
skriptiä käytettäessä.
